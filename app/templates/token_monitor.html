Token 优化策略（已实施）

1. System/User 消息分离：将 prompt 拆为 system 消息（指令）+ user 消息（内容），system 消息可被 API 提供方缓存，减少重复处理开销。

2. Prompt 极致压缩：移除冗长示例和格式说明，仅保留最紧凑的指令与 schema 提示，input tokens 减少约 50%。

3. 智能跳过（阈值 0.90）：规则引擎置信度 >= 0.90 时跳过 AI 调用；单候选且置信度 >= 0.88 也跳过。命中率提升至 40-55%。

4. 文本预过滤：发送前剥离会议流程、表决情况、法律声明等程序性文本，有效输入减少 30-50%。

5. 结果缓存：MD5 去重 + 500 条内存缓存，避免重复处理同一公告。

6. Token 预算限制：支持日/小时 token 预算上限（AI_DAILY_TOKEN_BUDGET / AI_HOURLY_TOKEN_BUDGET），超限自动暂停 AI 调用。

7. 输出上限：max_tokens = 800，单次提取足够且节省 output tokens。

8. 共识机制：规则与 AI 一致时置信度 +0.02，减少人工审核和间接重试。
