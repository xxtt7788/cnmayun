Token 优化策略（已实施）

1. 无信号跳过：标题+正文无人事变动信号时直接跳过 AI。消除约 44% 空返回调用。

2. 规则充足跳过（阈值 0.85）：规则置信度 >= 0.85 跳过；标题含姓名+事件类型且规则有候选也跳过。

3. 段落级提取：只发送含人事关键词的段落及上下文，长公告减少 60-80%。

4. 自适应截断：标题已有姓名+事件时，正文限 2000 字，避免发送全文。

5. 文本预过滤：剥离会议流程、表决、法律声明等，减少 30-50%。

6. System/User 分离：system 可被 API 缓存，减少重复处理。

7. 超短 JSON key：输出用 p/r/c/e/x 替代 person_name/role_raw 等，减少 output ~40%。

8. Prompt 极致压缩：system prompt ~90 tokens，移除 confidence 字段。

9. max_tokens=200：实际平均 output 仅 144，200 完全足够。

10. char_limit=4000：从 12000 降至 4000，input 减少 67%。

11. Token 预算 + 403 冷却：日/小时预算上限；403 冷却 1 小时；429 指数退避。

12. 缓存 + 固定置信度：MD5 去重缓存 500 条；固定 confidence=0.86。
